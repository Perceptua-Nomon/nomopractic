# nomopractic v0.1.0

**Released:** 2026-03-10
**Type:** Initial release

---

## Summary

First production release of the nomopractic Rust HAT daemon. Covers Phases 1–5:
IPC scaffold, I2C/ADC reads, PWM servo control with TTL watchdog, GPIO/MCU reset,
and deployment tooling. Verified live on a Pi Zero 2W / SunFounder PicarX.

---

## Features

### IPC Methods
| Method | Description |
|--------|-------------|
| `health` | Daemon liveness, schema version, uptime |
| `get_battery_voltage` | ADC channel A4 → voltage in volts |
| `set_servo_pulse_us` | Direct pulse-width command (500–2500 µs) |
| `set_servo_angle` | Angle-to-pulse conversion (0°–180°) |
| `read_adc` | Raw ADC read for any channel A0–A7 |
| `get_servo_status` | Active TTL lease table |
| `get_mcu_status` | Reset counter and elapsed time since last reset |
| `reset_mcu` | Assert/de-assert BCM5 (MCURST) for ≥ 10 ms |

### Safety
- Per-channel servo TTL lease watchdog — idles servo on expiry
- Client disconnect automatically releases all held leases

### Configuration
- TOML config file with env-var overrides (`NOMON_HAT_*`) and CLI `--config`
- IPC socket path, I2C bus/address, log level, TTL defaults all configurable

### Deployment
- Cross-compiled for `aarch64-unknown-linux-gnu` via `cross`
- `scripts/deploy.sh` — downloads binary from GitHub Releases, verifies SHA-256,
  atomically replaces `/usr/local/bin/nomopractic`, restarts systemd service
- Rollback: keeps previous binary as `nomopractic.bak`
