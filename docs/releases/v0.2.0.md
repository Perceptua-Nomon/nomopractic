# nomopractic v0.2.0

**Released:** 2026-03-11
**Type:** Feature release

---

## Summary

Feature release adding DC motor control support (Phase 6). Motors are controlled
via PWM on channels 12–15 using the TC1508S mode 1 protocol (independent PWM +
direction pins). All motors support a configurable TTL lease watchdog—if not
refreshed within the TTL, the daemon automatically idles the motor for safety.

Enhanced motor support extends nomon to wheeled platforms (PicarX, etc.) beyond
servo-only applications.

---

## Features

### New IPC Methods
| Method | Description |
|--------|-------------|
| `set_motor_speed` | Set motor speed (-100–100 %) with TTL lease |
| `stop_all_motors` | Immediately stop all motors and clear leases |
| `get_motor_status` | Query active motor TTL lease table |

### Configuration
- `MotorConfig` in TOML: `pwm_channel` (12–15), `dir_pin_bcm`, `reversed` flag
- PicarX defaults: Motor0 on PWM ch12/BCM24, Motor1 on PWM ch13/BCM23
- `motor_default_ttl_ms` global config option
- All options override-able via `NOMON_HAT_*` env vars

### Driver Details
- motor.rs: TC1508S protocol (PWM frequency 100 Hz, duty cycle via SET_CHANNEL_PULSE_US register)
- Direction pin via rppal GPIO (BCM numbers)
- Per-motor idle on lease expiry; no force-all-stop needed
- Client disconnect auto-stops held motors

### Bug Fixes
- **PWM channel stride bug (pre-existing):** Servo PWM was using `REG_CHN + channel*4`
  (pre-Phase-6); fixed to `REG_CHN + channel` (stride 1) to match SunFounder source.
- **PWM init bug (pre-existing):** Motor PWM timers (0–2) were not being initialized;
  fixed init_motor_pwm to handle all required channels.

## Breaking Changes

None. Motor IPC methods are purely additive; servo/GPIO/battery endpoints unchanged.

---

## Migration

Upgrade deployments by running `scripts/deploy.sh` or manually installing the
new binary:

```bash
cross build --target aarch64-unknown-linux-gnu --release
sudo systemctl stop nomopractic
sudo cp target/aarch64-unknown-linux-gnu/release/nomopractic /usr/local/bin/
sudo systemctl start nomopractic
```

For motor support, add motor configuration to `/etc/nomopractic/config.toml`:

```toml
motors = [
  { pwm_channel = 12, dir_pin_bcm = 24, reversed = false },
  { pwm_channel = 13, dir_pin_bcm = 23, reversed = false },
]
motor_default_ttl_ms = 2000
```

---

## Dependencies

No new dependencies. Phase 6 uses only `rppal` (already present for GPIO/I2C).

---

## Known Limitations

- Motor control is synchronous on the daemon side (no async PWM hardware).
- TTL lease minimum granularity is daemon poll frequency (adjustable via `watchdog_poll_ms`).
